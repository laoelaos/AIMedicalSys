# 设计审查报告（v5 r2）

## 审查结果
REJECTED

## 发现

- **[一般]** `keySet()` 接口描述与实现契约矛盾：SessionStore 公开接口（第42行）声明 `keySet()` 返回"不可变快照集合"，但方法行为契约（第159行）明确标注"非不可变快照，迭代器为弱一致"。上一轮 r1 审查已将此列为 [一般] 问题并制定了修正措施，但第42行的接口描述未被同步更新，导致文档内部仍存在自相矛盾。该矛盾可能在 Phase 5 RedisStore 替换时引发实现歧义：开发者参照接口描述可能返回不可变快照，而 ConcurrentHashMapStore 实际返回实时视图，两者语义不一致。

## 修改要求
修正 SessionStore 公开接口第42行 `keySet()` 描述，使其与方法行为契约保持一致：将"返回所有键的不可变快照集合"改为"返回所有键的 Set 视图（由 ConcurrentHashMap 支撑），迭代器为弱一致，不提供快照隔离保证"。
