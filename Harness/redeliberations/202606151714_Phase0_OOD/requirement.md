# Phase 0 OOD 设计需求

## 任务目标
产出一份核心OOD设计文档，目标是实现智慧云脑诊疗平台 roadmap 中的 Phase 0（最小化骨架/项目初始化阶段）。

## 设计范围
Phase 0 不实现任何业务功能，只交付协作所需的最小骨架。设计须覆盖以下维度：

### 1. 共享工程结构
- Monorepo 布局设计
- 多模块后端工程结构（患者/医生/管理员/公共模块/AI能力模块等）
- 多端前端工程结构（患者端/医生端/管理员端）
- 支持多人并行贡献的模块划分

### 2. 接口契约框架
- 统一响应包装（Result<T> 泛型类）
- 分页请求/响应规范
- 错误码命名空间设计
- AI 能力占位类型与接口定义
- 全局异常处理机制

### 3. 数据与权限模型骨架
- 数据实体基类（公共字段如 id、createdAt、updatedAt 等）
- 权限模型：角色—岗位—功能三级权限模型
- 用户/角色/岗位基础实体定义

### 4. 协作规范（架构层面）
- 包命名规范与分层约定
- 模块间依赖关系与调用规则
- API 版本管理策略

### 5. 本地开发体验（架构层面）
- 统一配置管理设计
- 多模块构建依赖设计

### 6. 持续集成占位（架构层面）
- 构建流水线中的模块依赖关系

### 7. AI 能力模块 Mock 占位
- AI 能力模块接口契约定义
- Mock 数据占位架构
- 降级策略框架

## 设计产出要求
- 必须是一份可直接指导编码实现的 OOD 设计文档
- 包含：整体架构视图、模块分解图/说明、包结构设计、关键类/接口定义、数据实体基类设计、权限模型设计、AI 能力契约定义
- 异常场景和边界条件已考虑
- 设计可直接指导编码实现
