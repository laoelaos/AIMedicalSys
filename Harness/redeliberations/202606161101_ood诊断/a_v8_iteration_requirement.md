## 当前审查结果

### 1. §2 诊断项分类体系存在系统性矛盾（严重）
§2（偏离需求文档）下全部5个诊断项（§2.1~§2.5）均在其"降级依据"段落中明确声明"不影响 Phase 0 三项验收标准"、"属于 Phase 0→Phase N 衔接准备项"，却被统一归类在"偏离需求文档"标题下，构成分类矛盾。修复者面临"修还是不修"的决策困境。
- **改进建议**：将明确属于"Phase 1+ 衔接准备"的项（§2.1、§2.3、§2.4、§2.5）移入独立章节（如新增§8 跨阶段衔接建议），或在每项标题后标注"（衔接建议，非Phase 0偏离）"。保留在§2下的应是真正阻塞 Phase 0 验收标准或在 Phase 0 范围内构成明显偏离的项。

### 2. §2.2 权限矩阵诊断中"线下接诊医生角色缺失"的有效性不足（一般）
将 Phase 1 的数据播种任务缺失诊断为 Phase 0 OOD 的"偏离需求文档"，混淆了"OOD 文档设计框架是否预留建模能力"与"Phase 1 是否已完成角色数据初始化"两个不同阶段的责任边界。
- **改进建议**：调整为"OOD 权限模型框架已预留全部角色/岗位的建模能力，建议在§3.3 末尾补充对 Phase 1 数据种子中角色/岗位枚举的指引说明"，或将该项从"偏离需求文档"降级为"Phase 1+ 衔接提示"。

### 3. §2.4 修复建议存在"伪二选一"困境，修复者无法直接行动（一般）
修复者须知为实体阶段映射表提供了两种路径，但路径 A 被报告自身分析立即否定（"违背 OOD 最小化修改原则"），路径 B 附加了前置条件（"需项目管理者确认"），修复者无法在当前条件下直接行动。
- **改进建议**：（1）如果路径 A 不可行，直接推荐路径 B 并明确标注该映射表不属于 Phase 0 OOD 修改范围；（2）如果两条路径均可接受，则移除对路径 A 的否定性评价并给出明确推荐。

### 4. §3.2 对 SecurityConfig 超前实现的风险提示与 §4.1 CSRF 修复项之间的优先级倒挂（一般）
SecurityConfig 超前实现中 PasswordEncoder 配置错误风险（隐蔽、影响大）仅作为风险提示不设优先级，而 CSRF 缺失（易发现、易修复）标记为 P1，两者不成比例。
- **改进建议**：（1）在§3.2 风险提示中为 PasswordEncoder 配置错误补充优先级标注（至少 P2，建议 P1）；（2）或在§4.1 中解释为何 CSRF 的 P1 优先级高于 PasswordEncoder 配置错误，让读者理解两处优先级差异的权衡依据。

### 5. §3.1 ADR 建议的边际价值不明确（轻微）
建议补充 ADR 说明超前定义 DTO 的理由，但 OOD 已有隐含覆盖，新增维护成本可能超过收益。
- **改进建议**：降级为可选建议，或将内容合并到现有 §7 设计决策表相关备注字段，不另立独立 ADR。同时明确标注该建议为"可选的文档补充项，不影响 Phase 0 交付质量"。
