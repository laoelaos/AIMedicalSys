## 需要解决的问题（仅 RETRY 时存在）

- **问题描述**：§2.2 权限矩阵缺口被赋予 P0 优先级，与 Phase 0 验收标准不匹配，不阻塞骨架验收却标记为最高优先级

- **严重程度**：高
- **改进建议**：将 §2.2 降级为 P1，属 Phase 0→Phase 1 衔接准备项而非 Phase 0 阻塞项

- **问题描述**：§2.1（P1）与 §2.2（P0）之间存在可验证的优先级倒挂，§2.1 错误码命名约定实际阻塞程度等于或高于 §2.2
- **所在位置**：b_v2_diag_v1.md §2.1 vs §2.2
- **严重程度**：一般
- **改进建议**：将 §2.1 升级为 P0 或将两者统一为 P1 并明确各自时间约束

- **问题描述**：§3.2 将容器化部署脚本和本地代码质量检查工具等 DevOps 工程配置项列为 OOD 偏离问题，超出架构设计文档的诊断边界，且与 §3.1 自身边界定义矛盾
- **所在位置**：b_v2_diag_v1.md §3.2
- **严重程度**：一般
- **改进建议**：从"偏离"类别移除 §3.2，或调整为"项目管理提醒"而非"OOD 偏离问题"

- **问题描述**：§4.1 将 Phase 0 全量 DTO 定义（路线图要求的"接口契约框架"核心交付物）定性为"需求稳定性风险"，判断标准与 §4.2 同类情况相反
- **所在位置**：b_v2_diag_v1.md §4.1
- **严重程度**：一般
- **改进建议**：将 §4.1 从"问题"降级为"补充说明"级别的设计笔记，与 §4.2 判断标准保持一致
