## 二、产出中的质量问题

### 问题 1：§2.2 子问题分类混合，修复者难以统一判断行动指向（一般）

- **位置**：`a_v8_diag_v1.md` §2.2，第 44–49 行
- **问题描述**：§2.2 以"偏离需求文档"标题呈现，但其内部 4 条子问题的性质不一致：
  - 子点 1（角色权限边界未完全映射）和子点 4（DataPermissionEvaluator 接口担忧）是真实的"需求已明确定义→OOD 未贯穿"的偏离问题；
  - 子点 2（线下接诊医生角色缺失）已被报告自身修订为"Phase 1 数据种子任务，非 Phase 0 OOD 设计缺失"；
  - 子点 3（药房医生跨模块授权路径）被标注为"Phase 1+ 视角"。
  
  两条偏偏离类子点与两条非偏离类子点并列在同一诊断项下，读者在"这条是否需修改 OOD"的判断上需要逐条过滤。降级依据段落以"该缺口属于 Phase 0→Phase 1 衔接准备项"统一定性，无法对应到子点 1 和子点 4 的真实偏离性质。修复者在拿到报告后需要自行拆分判断 4 个子点各自是否属于 OOD 修改范围。

- **严重程度**：一般
- **改进建议**：两种可选方案——（A）将子点 2 和子点 3 移入 §8（跨阶段衔接建议），保留子点 1 和子点 4 在 §2.2 作为纯偏离诊断项，标题相应调整为"权限矩阵角色映射缺漏与接口扩展性关注"等聚焦表述；（B）若保留当前结构，在 §2.2 开头增加一句话声明，如："本项混合了两类内容：子点 1、子点 4 属于 OOD 应补充的真实偏离；子点 2、子点 3 属于 Phase 1+ 衔接准备，OOD 无需修改但需在后续阶段纳入规划。"

### 问题 2：§3.2 PasswordEncoder 测试建议缺少 Profile 激活前提（一般）

- **位置**：`a_v8_diag_v1.md` §3.2，第 77 行
- **问题描述**：建议提出"在 Phase 0 中添加独立的 PasswordEncoder 单元测试……以验证 Bean 创建和基本编码/匹配功能正常"。但 §3.2 上文已说明 PasswordEncoder 属于 `@Profile("!phase0")` 配置——该 Bean 在 Phase 0 默认 Profile 下不会被创建。若修复者直接按照建议尝试添加单元测试，会发现无法注入 PasswordEncoder Bean，需要额外了解 Profile 激活机制才能执行。建议缺少 Profile 激活这一执行前提，使推荐路径的可直接执行性打了折扣。
- **严重程度**：一般
- **改进建议**：补充测试方法中的 Profile 激活说明，例如："注意 PasswordEncoder Bean 位于 `@Profile("!phase0")` 配置下（不属于 Phase 0 SecurityConfig 默认激活范围），单元测试需通过 `@ActiveProfiles("phase1")` 或其他方式激活对应 Profile，方可注入并验证 Bean 的创建和编码/匹配功能。"

---

### 问题 3：§5 补充说明子节结构松散，信息密度偏低（轻微）

- **位置**：`a_v8_diag_v1.md` §5，第 109–131 行
- **问题描述**：§5 分 A–E 共 5 个子节。其中 §5.B（Phase 0 边界自洽性）的内容已在 §7 偏离路线图声明和 §3 反向边界检查中覆盖；§5.C（ASCII 箭头语义说明）的内容已在 §6 逻辑错误声明中覆盖其结论。这两项以独立子节形式重复呈现，使得补充说明区的信息稀释。§5.D（容器化部署脚本与 lint 工具配置）和 §5.E（协作规范缺失）虽都标为"项目管理提醒"但分布在两个子节，可合并为单一"项目管理提醒"子节。整体阅读体验上，5 个子节中有 3 个是已有结论的复述或逻辑上可合并的相邻项。
- **严重程度**：轻微
- **改进建议**：将 §5.B 和 §5.C 合并为一条脚注式说明加在 §6 或 §7 末尾，将 §5.D 和 §5.E 合并为一条"项目管理提醒"。保留 §5.A 作为独立的补充说明（其内容涉及 §1.1 的依赖合理性论证，具有独特信息价值）。

## 五、优先级排序合理性

- **P0 项**：无。与路线图验收标准一致（报告明确指出无项阻塞三项验收标准）。
- **P1 项**：§1.1（文档矛盾→影响维护者理解）、§2.1（错误码映射缺失→影响 Phase 1+ 对齐）、§2.2（权限矩阵缺口→影响 Phase 1+ 对齐）、§2.3（DTO 约束缺失→影响 Phase 1+ 对齐）、§3.2（PasswordEncoder 隐藏风险→Phase 1 认证启用时暴露）、§4.1（CSRF 基线缺失→Phase 1 开发者遗忘风险）、§8.1（实体阶段映射缺失→Phase 1 衔接）。
- **P2 项**：§1.2（JDBC scope 风格不一致→实际影响极低）、§4.2（@SQLRestriction 限制说明→Phase 2+ 影响）、§8.2（超时阈值对齐→Phase 2+ 信息补充）。

**评估**：P1 项数量偏多（7 项），但均符合其定义的"影响 Phase 1+ 衔接质量"标准，分配合理。无明显的优先级倒挂或双标准。
