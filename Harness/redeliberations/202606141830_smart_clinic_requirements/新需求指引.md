# 需求分析文档——编写指引

需求分析文档应当满足以下标准：

## 应当包含的内容
- 业务背景与目标
- 角色定义（谁使用系统，做什么）
- 功能清单（用户需要什么能力，而非系统的内部模块）
- 用户场景（用户如何使用这些功能完成业务目标）
- 业务流程（端到端的业务流转，以角色为主语描述）
- 非功能要求（性能、可用性、安全性等，用业务语言描述）
- 验收标准（可验证的业务验收条件）

## 不应包含的内容
- 后端接口路径、方法名、请求/响应字段
- 前端组件、路由、状态管理方案
- 数据库表结构、字段类型、ER图
- 状态机流转图、状态枚举值
- API契约、参数校验规则
- 超时阈值、并发数等实现级指标（除非是明确的业务SLA）
- AI模型调用方式、服务封装方式
- 内部模块拆分、层间调用关系

## 核心原则
- 用**业务语言**描述，让非技术人员也能理解
- 关注**做什么（What）**，而非**怎么做（How）**
- 每个需求条目都是**可验收的**
- 角色场景驱动，而非数据驱动
