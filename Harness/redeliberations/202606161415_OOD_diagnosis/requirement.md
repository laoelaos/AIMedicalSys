# 问题定位任务：OOD 文档诊断

## 任务目标
对 `Docs/04_ood_phase0.md`（Phase 0 最小化骨架 OOD 设计方案）进行全面的问题定位诊断，识别文档中存在的各类问题。

## 检查维度
1. **定义矛盾（定义矛盾）**：文档内部或与其他文档之间存在定义不一致、自相矛盾的问题
2. **事实错误（事实错误）**：技术事实、版本号、依赖关系等方面的陈述性错误
3. **逻辑错误（逻辑错误）**：推理链条断裂、前提与结论不匹配、遗漏关键因果关系
4. **偏离路线图 Phase 0 阶段（偏离路线图phase0阶段）**：OOD 设计内容与 `Docs/03_roadmap.md` 中 Phase 0 的范围、目标、约束不一致的地方（包括超出 Phase 0 范围的内容，或遗漏 Phase 0 应包含的内容）

## 参考文档
- `Docs/04_ood_phase0.md`：被诊断的 OOD 设计文档
- `Docs/03_roadmap.md`：项目实现路线图，用于对照检查偏离情况

## 输出要求
诊断报告应列出每个问题的以下信息：
- 问题描述
- 所在位置（章节/行号范围）
- 严重程度（严重/中等/轻微）
- 改进建议
