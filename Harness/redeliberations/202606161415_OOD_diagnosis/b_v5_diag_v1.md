## 问题清单

### 问题A（中等）：Problem 4推荐方案的"方案A"初始定义与推荐组合"方案A+子方案A2"存在结构矛盾

**所在位置**：
- 第115–118行：方案A定义"将13项能力的DTO从完整的字段级定义缩减为**仅保留类声明+默认构造器（class 骨架，不含任何字段）**"
- 第126–128行：子方案A2"为Phase 0**保留少量关键字段**作为'Phase 0 Mock 字段子集'"
- 第163行：结论"推荐**方案A + 子方案A2**"

**问题描述**：
方案A的基础定义以"不含任何字段"为前提，但子方案A2要求"保留少量关键字段"。两套前提存在直接矛盾。结论将二者组合为"方案A+子方案A2"，执行者面对这一组合时难以理解最终预期状态：
- 究竟是按方案A把DTO剪成零字段的空壳？
- 还是按A2保留部分字段？
- 还是"零字段空壳的基础上再挑出部分字段保留"——这本身在语义上不自洽（零字段空壳上无法"保留"字段）。

虽然第163行的结论文本通过描述性语言（"保留空壳DTO类声明+保留核心字段"）在语义层面做了弥补，但方案框架的结构矛盾仍未解决：方案A的初始描述与推荐组合在逻辑前提上冲突。

**该问题经历了v3→v6共4轮修订尚未被发现**，此前内部审议聚焦于编译期约束分析（是否有）、Mock冲突分析（是否有）、选优标准（是否缺）、依赖关系方向（是否一致）等维度，未发现推荐方案层级结构本身的自相矛盾。

**严重程度**：中等

**改进建议**：

方案A第117行的"不含任何字段"应删除，或明确表述为"保留DTO类声明，仅按子方案策略选择性保留字段"——即方案A本身不预设"零字段"前提，而是作为"缩减式保留"的上层框架，其下A1（零字段）、A2（保留关键字段子集）、A3（反射解耦）为互斥的子选项。

修改后示例：
```
方案A：保留所有DTO类型定义但缩减为"空壳占位"（仅保留类声明与子选项指定的字段）
```
并在方案描述中删除"不含任何字段"的绝对化表述。
