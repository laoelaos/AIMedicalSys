## 发现的问题

### 1. [轻微] Issue 2 操作顺序建议中交叉引用未按清理方案限定范围

- **所在位置**：Line 424，`Issue 2` → `生产脏数据行动指引` → `操作顺序建议`
- **问题描述**：操作顺序建议中第②步"业务决策选择清理方案"与第③步"执行数据清理"之间插入了无条件警告"⚠ 执行数据清理前，必须确认 Issue 4 的 enabled 清理已完成"。该警告仅对方案 B（批量置为不可用，`SET enabled = 0 WHERE password IS NULL`）是必要的前提条件。对于方案 A（人工补录）和方案 C（直接清理），不需执行 `SET enabled = 0`，故也不存在与 Issue 4 的时序依赖。无条件警告可能导致选择方案 A/C 的执行者产生困惑：为何自己不需要关心 Issue 4 清理，但文档又说"必须确认"。
- **严重程度**：轻微
- **改进建议**：将警告限定于方案 B 场景，例如改为"⚠ 若选择方案 B（批量禁用），执行数据清理前必须确认 Issue 4 的 enabled 清理已完成……详见下节"。或保持现有警告，在紧接其后的"与 Issue 4 清理策略的交叉数据冲突"小节首句补充说明"本冲突仅适用于选择方案 B 的场景，方案 A/C 不涉及此依赖"。
