### 2. [中等] Issue 4 缺少修复路径的权衡分析

**问题描述：**
`enabled`/`visible` 布尔字段缺少默认值问题至少存在三个可行修复路径：
1. Java 端添加字段默认值 `private Boolean enabled = true;`
2. 使用 `@ColumnDefault("1")` 或 `@Column(columnDefinition = ...)` 注解
3. 仅修改 DDL 加 `NOT NULL`（依赖数据库 DEFAULT 触发机制）

这三个方案的副作用完全不同：
- 方案 1 最直接，但不会修复已存在的 NULL 数据库记录
- 方案 2 对 Hibernate 以外的 SQL 操作无保护
- 方案 3 单独使用会因 Hibernate INSERT 包含所有列而不生效（报告自身已分析此点）

报告只描述了问题现象和根因，没有对比这些方案的优劣，也没有给出倾向性推荐。执行者拿到报告后不知道"该选哪种方式修"，与报告自述的"修复者可直接行动"粒度存在差距。

**所在位置：** Issue 4 章节（第 113-148 行）

**严重程度：** 中等

**改进建议：** 对比上述三个修复路径的优缺点（包括对已有数据的影响、对 Hibernate DDL auto 行为的影响、对数据库可移植性的影响），给出推荐方案及其理由。

**所在位置：** Issue 2 "影响范围"（第 56-60 行）

**严重程度：** 中等
