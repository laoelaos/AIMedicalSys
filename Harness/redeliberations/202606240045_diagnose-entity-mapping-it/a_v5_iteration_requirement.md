### 问题 1 [严重] — @SQLRestriction 行为评判事实错误，导致 Issue 3 影响评估和优先级排序依据失准

**所在位置**：Issue 3「影响范围」第 247 行；「优先级排序」交叉对比分析第 351-354 行

**问题描述**：报告第 353 行声称 "`deleted IS NULL` 和 `deleted = 0` 在 `@SQLRestriction("deleted = false")` 过滤下均被视为'未删除'"。这是事实错误。`BaseEntity.java:24` 声明 `@SQLRestriction("deleted = false")`，Hibernate 追加 `WHERE deleted = false` 到所有查询。在 SQL 三值逻辑中，`NULL = 0` 返回 NULL（unknown），而非 TRUE。WHERE 子句将 NULL 视为 false，因此 `deleted IS NULL` 的记录会被过滤掉——实际被当作"已删除"处理。

**改进建议**：
1. 修正该错误陈述，明确 `deleted IS NULL` 在 `@SQLRestriction` 下的实际行为（被过滤）；
2. 重新评估 Issue 3 的业务影响等级——脏数据记录将被静默"隐藏"，影响范围可能超出当前描述；
3. 基于修正后的影响分析，重新审视交叉对比表格中的"业务影响"和"运行时异常风险"评分；
4. 若重新评估后 Issue 3 的优先级需调整，同步更新时序依赖图和修复顺序建议。
