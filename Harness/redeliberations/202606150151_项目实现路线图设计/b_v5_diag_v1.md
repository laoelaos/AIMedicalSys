## 2. v5 引入 / 残留的新问题

### 2.1 [严重] Phase 3.5 与 Phase 4.5 包命名混淆（D1/D2 在两阶段含义不同）

- **问题描述**：v5 修订说明 P1-C L668 引入"Phase 3 包 D1/D2 = AI 处方审核 / AI 病历生成"；但 Phase 4.5 自 v4 起已有"包 D1/D2 = 药库基础管理 / 调拨管理"。两阶段使用相同字母标识但完全不同的内容。读者跨阶段追踪任务包时极易混淆（例如，跨阶段会议中提到"包 D1"无法判断指代）。
- **所在位置**：Phase 3.5 L247-248（包 D1/D2 = AI 能力）；Phase 4.5 L340-341（包 D1/D2 = 药库/调拨）。
- **严重程度**：一般（不阻碍阅读，但降低可追踪性，且在并行沟通时易出错）。
- **改进建议**：
  - 方案 A：Phase 3.5 重命名为包 D-AI1 / 包 D-AI2（明确 AI 能力归属），与 Phase 4.5 的 D1/D2 解耦。
  - 方案 B：Phase 3.5 维持包 D1/D2，但要求所有阶段间引用必须显式带阶段前缀（如 "Phase 3 包 D1"），并在协作硬约束表中加注此约定。
- **独立结论**：P1-C 拆分本身正确，但**拆分方式未与 Phase 4.5 既有命名约定对齐**，v5 自检中未提及。
